# Contribuindo

Contribuições devem seguir as diretrizes do DACC/UNIR e boas práticas de desenvolvimento.

## Antes de contribuir

1. Leia a documentação disponível no repositório.
2. Verifique se já existe issue relacionada ao problema ou melhoria.
3. Caso necessário, abra uma issue descrevendo contexto, motivação e proposta.
4. Envie um pull request com descrição clara e objetiva.

## Boas práticas

- Mantenha o código legível e organizado.
- Escreva commits com mensagens claras.
- Evite alterações não relacionadas ao objetivo do pull request.
- Atualize a documentação quando houver mudança de comportamento, estrutura ou execução.
- Teste localmente antes de abrir o pull request.

## Pull requests

Ao abrir um pull request, procure informar:

- o que foi alterado
- por que a alteração foi necessária
- como validar a mudança
- impactos conhecidos

## Padrão esperado

- respeito à estrutura do projeto
- consistência com a stack adotada
- documentação mínima das mudanças
- aderência às decisões técnicas já existentes, salvo justificativa explícita

## Contexto institucional

Este projeto está vinculado ao Departamento Acadêmico de Ciência da Computação da Universidade Federal de Rondônia (UNIR). Alterações devem preservar a clareza, a rastreabilidade e a utilidade acadêmica e técnica do repositório.
